Tech Assist AI Chatbot

Enterprise AI Solution 8 min read

The Problem: Long Wait Times, Zero Visibility

At Macquarie Group, employees visiting Tech Assist kiosks faced 10-15 minute wait times for routine issues like password resets, VPN troubleshooting, and software downloads. There was no queue visibility, phone support had indefinite hold times, and Tech Assist teams were overwhelmed with repetitive queries that could be automated. This resulted in over 2,500 employee hours wasted monthly and significant productivity loss across the organisation.

The core issue was clear: employees needed instant, 24/7 access to tech support for common questions, while Tech Assist needed to focus their expertise on complex technical problems that truly required human intervention.

The Solution: AI-Powered Self-Service Support

I designed and built Tech Assist AI (TAAI), an intelligent chatbot powered by Azure OpenAI GPT-4 that provides instant tech support by searching internal knowledge bases (Confluence and SharePoint). Using retrieval-augmented generation (RAG) architecture, TAAI answers questions in under 3 seconds with source citations, reducing wait times from minutes to near-zero.

How it works:

  • User asks a question: "How do I reset my VPN password?"
  • System searches: Hybrid vector + keyword search across 10,000+ internal documents
  • AI generates response: GPT-4 creates contextual answer using retrieved documentation
  • User receives answer: Response with source citations in <3 seconds
  • Smart escalation: If confidence is low or issue is complex, seamlessly hands off to Tech Assist

Key Capabilities:

⚡ Instant answers with <3 second response time
🌐 24/7 availability via web, Microsoft Teams, and SharePoint
🧠 RAG pipeline with hybrid search (vector + keyword)
📚 Source citations from Confluence and SharePoint docs
👤 Smart escalation to human agents when needed
📊 Live queue tracking for Tech Assist kiosk wait times

Technical Architecture: Building for Scale and Accuracy

The core challenge was ensuring accurate, trustworthy responses at scale. I architected a comprehensive RAG pipeline that prioritises retrieval quality and response reliability:

RAG Pipeline:

  • Query Processing: User input is preprocessed and embedded using Azure OpenAI's text-embedding-ada-002 model
  • Hybrid Search: Azure Cognitive Search performs both dense vector search (semantic similarity) and sparse keyword search (BM25), then merges results using reciprocal rank fusion
  • Context Building: Top 5 documents are retrieved with metadata (source, category, date) and chunked at 512 tokens with 50-token overlap
  • Response Generation: GPT-4 generates answers using retrieved context with strict prompt engineering to prevent hallucinations
  • Confidence Scoring: System calculates confidence based on retrieval quality and response completeness, triggering escalation if low
  • Source Attribution: Every response includes citations linking back to source documents

Tech Stack:

  • Backend: FastAPI (Python 3.11), LangChain for RAG orchestration
  • AI/ML: Azure OpenAI (GPT-4 for generation, text-embedding-ada-002 for embeddings)
  • Data: Azure Cognitive Search (vector store), PostgreSQL (conversation history), Redis (caching)
  • Frontend: React 18 + TypeScript, Tailwind CSS, WebSocket for real-time chat
  • Infrastructure: Azure App Service, Azure Key Vault (secrets), Application Insights (monitoring)

Beyond cost savings, TAAI provides data insights on common tech issues, enables continuous knowledge base improvement, and frees Tech Assist teams to focus on complex problems requiring human expertise.

Technical Challenges & Solutions

1. Ensuring Answer Accuracy (Preventing Hallucinations) RAG systems can hallucinate if not properly constrained. I implemented strict prompt engineering instructing GPT-4 to only use provided context, added confidence scoring based on retrieval quality and response completeness, and designed clear escalation paths when confidence falls below 70%. Every response includes source citations so users can verify information, building trust through transparency.
2. Handling Knowledge Base Scale (10,000+ Documents) With 10,000+ documents from Confluence and SharePoint, efficient retrieval was critical. I designed a hybrid search approach combining dense vector search (semantic similarity) with sparse keyword search (BM25), then used reciprocal rank fusion to merge results. This ensured both semantic understanding and exact keyword matching, dramatically improving retrieval accuracy.
3. Balancing Performance and Cost GPT-4 API calls cost approximately $0.03 per query at scale. I implemented Redis caching for frequent queries (reducing redundant API calls by 40%), set conservative token limits (500 max response tokens), and used query classification to route simple questions to cached responses. This reduced monthly costs from $3,800 to $2,270 while maintaining quality.
4. Building User Trust and Adoption Employees needed to trust TAAI before adopting it. I prioritised transparency: showing confidence scores, providing source citations, making escalation easy (one-click handoff to Tech Assist), and collecting feedback after every interaction. The marketing campaign emphasised that TAAI complements (not replaces) human Tech Assist teams, reducing resistance to adoption.

Key Learnings

1. RAG Quality = Data Quality The system is only as good as its knowledge base. I invested significant time in data processing: cleaning HTML from Confluence exports, chunking documents at logical boundaries (512 tokens with 50-token overlap for context continuity), and enriching chunks with metadata (source, category, last updated date). This upstream work directly improved downstream retrieval accuracy and response quality.
2. User Experience Drives Adoption Technical excellence isn't enough—users need a seamless experience. I designed multiple access points (web app, Teams bot, SharePoint widget), ensured <3 second response times, added real-time typing indicators for perceived performance, and created intuitive escalation flows. The embedded SharePoint widget drove 40% of initial traffic by meeting users where they already work.
3. Change Management is Critical for AI Adoption AI adoption requires more than building great technology—it requires organisational buy-in. I developed a comprehensive change management plan: securing executive sponsorship, creating physical marketing at pain points (kiosks), developing training materials for Tech Assist teams on when/how to use TAAI, and running a pilot program to build internal champions. The multi-channel marketing strategy was as important as the technical implementation.
4. Monitoring Enables Continuous Improvement AI systems need active maintenance and iteration. I built analytics dashboards tracking resolution rate (% of queries resolved without escalation), escalation reasons (identifying knowledge gaps), low-confidence queries (highlighting areas for prompt improvement), and user feedback sentiment. This data loop enabled weekly knowledge base updates, monthly prompt engineering refinements, and quarterly feature releases based on actual usage patterns.
